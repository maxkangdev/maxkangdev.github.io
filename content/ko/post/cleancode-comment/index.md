## 주석이 꼭 필요한가? 

**필요없다.**   

대부분의 주석은 실패한 코드를 만회하기 위해 사용된다.  
>이해하기 쉬운 코드를 짜지 못하니 주석으로 장황하게 설명하는것이다. 

주석은 오래될 수록 코드와 멀어진다. 
> 코드가 변경될때 마다 관련된 주석을 모두 찾아서 매번 업데이트 해주나? 사실상 불가능하다.  
> 코드와 불일치하는 주석은 아예 없는 주석보다 훨씬 안좋다.

진실은 한 곳에 존재한다
> 바로 코드다.  
> 주석 쓸 시간에 좋은 코드를 만드는데 시간을 써라.  
> 코드를 읽는 것만으로 의도를 파악할 수 있도록 말이다.


하지만 주석을 아예 안쓰기는 현실적으로 불가능하다. 잘 달린 주석은 그 어떤 정보보다 유용하다.   
좋은 주석과 나쁜주석을 구별해보자. 

## 좋은 주석 
아래는 써도 나쁘지 않은 주석들이다.  
**하지만 가장 좋은 주석은 필요 없는 주석이다!** 
### 법적인 주석
> 법적 요구사항이나 저작권 정보, 라이선스 정보 등을 명시하기 위한 주석입니다. 일반적으로 파일의 맨 위에 위치합니다.

**예제**
```java
// Copyright (C) 2023 by Robert C. Martin. All rights reserved.
// Released under the MIT License.
```

### 정보를 제공하는 주석
> 코드의 특정 부분에 대한 추가 정보를 제공하기 위한 주석입니다. 예를 들어, 추상화된 메서드의 구현 세부사항을 설명할 때 사용됩니다.

**예제**
```java
// 이 메서드는 HTTP GET 요청을 보내고 응답을 반환합니다.
public HttpResponse sendGetRequest(String url) {
    // ...
}
```

### 의도를 설명하는 주석
> 개발자가 코드를 작성한 의도나 이유를 설명하는 주석입니다. 특정 알고리즘을 선택한 이유나 비즈니스 로직의 배경을 설명할 때 유용합니다.

**예제**
```java
// 성능을 위해 이진 탐색 알고리즘을 사용했습니다.
int index = Arrays.binarySearch(sortedArray, target);
```

### 의미를 명료하게 밝히는 주석
> 코드의 의미를 명확히 하기 위해 사용하는 주석입니다. 특히 복잡한 로직이나 매직 넘버(magic number)를 설명할 때 유용합니다.

**예제**
```java
// 86400은 하루의 초 단위 시간입니다.
int secondsInADay = 86400;
```

### 결과를 경고하는 주석
> 특정 코드를 실행할 때 발생할 수 있는 부작용이나 위험을 경고하기 위한 주석입니다.

**예제**
```java
// 이 메서드는 데이터베이스 연결을 닫습니다. 재사용 전에 다시 연결해야 합니다.
public void closeDatabaseConnection() {
    // ...
}
```

### Todo 주석
> 나중에 수정하거나 추가해야 할 작업을 표시하기 위한 주석입니다. 임시로 작성한 코드나 미완성 기능을 표시할 때 사용됩니다.

**예제**
```java
// TODO: 예외 처리를 추가해야 합니다.
public void processData() {
    // ...
}
```

### 중요성을 강조하는 주석
> 특정 코드가 중요한 이유를 강조하거나, 특정 조건에서만 실행되어야 함을 설명하는 주석입니다.

**예제**
```java
// 이 코드는 시스템 성능에 큰 영향을 미치므로 주의해서 수정해야 합니다.
public void optimizeSystemPerformance() {
    // ...
}
```

### 공개 API에서 Javadocs
> 공개 API(Public API)를 문서화하기 위해 사용되는 주석입니다. Javadoc 형식으로 작성되며, 클래스, 메서드, 파라미터, 반환 값 등을 설명합니다.

**예제**
```java
/**
 * 두 정수를 더한 결과를 반환합니다.
 * 
 * @param a 첫 번째 정수
 * @param b 두 번째 정수
 * @return 두 정수의 합
 */
public int add(int a, int b) {
    return a + b;
}
```


## 나쁜 주석 
코드의 가독성을 해치거나, 불필요한 정보를 제공하거나, 오히려 혼란을 초래하는 주석들이다. 
대다수가 이런 주석들이니 왠만하면 쓰지말자

### 주절거리는 주석
> 의미 없는 내용이나 불필요한 설명으로 가득한 주석입니다. 코드를 이해하는 데 도움이 되지 않습니다.

**예제**
```java
// 이 메서드는 뭔가를 합니다. 아마도 데이터를 처리할 겁니다. 잘 모르겠네요.
public void doSomething() {
    // ...
}
```

### 같은 이야기를 중복하는 주석
> 코드가 이미 명확한데도 같은 내용을 반복하여 설명하는 주석입니다.

**예제**
```java
// x에 1을 더합니다.
x = x + 1;
```

### 오해할 여지가 있는 주석
> 코드의 실제 동작과 일치하지 않거나 오해를 불러일으킬 수 있는 주석입니다.

**예제**
```java
// 이 메서드는 항상 true를 반환합니다. (하지만 실제로는 조건에 따라 false를 반환할 수 있음)
public boolean isAlwaysTrue() {
    return condition ? true : false;
}
```

### 의무적으로 다는 주석
> 규칙이나 표준 때문에 의미 없이 추가된 주석입니다.

**예제**
```java
// 기본 생성자
public MyClass() {
    // ...
}
```

### 이력을 기록하는 주석
> 버전 관리 시스템(예: Git)으로 관리할 수 있는 내용을 주석으로 기록하는 경우입니다.

**예제**
```java
// 2023-10-01: John Doe - 초기 구현
// 2023-10-05: Jane Smith - 버그 수정
public void oldMethod() {
    // ...
}
```

### 있으나 마나 한 주석
> 아무런 가치가 없거나 이미 코드로 충분히 설명 가능한 내용을 담은 주석입니다.

**예제**
```java
// 반복문 시작
for (int i = 0; i < 10; i++) {
    // ...
}
```

### 무서운 잡음
> 불필요한 주석으로 인해 코드의 가독성을 해치는 경우입니다.

**예제**
```java
/////////////////////////////
// 이 부분은 중요합니다! //
/////////////////////////////
public void importantMethod() {
    // ...
}
```

### 함수나 변수로 표현할 수 있는 주석
> 주석 대신 함수나 변수의 이름으로 의도를 표현할 수 있는 경우입니다.

**예제**
```java
// x가 0보다 큰지 확인합니다.
if (x > 0) {
    // ...
}

// 개선: 함수나 변수 이름으로 표현
if (isPositive(x)) {
    // ...
}
```

### 닫는 괄호에 다는 주석
> 중첩된 코드의 닫는 괄호에 주석을 추가하는 경우로, 코드를 작은 함수로 분리하는 것이 더 나은 해결책입니다.

**예제**
```java
while (condition) {
    // ...
} // end while
```

### 공로를 돌리거나 저자를 표시하는 주석
> 코드의 저자나 수정자를 표시하는 주석으로, 버전 관리 시스템으로 대체 가능합니다.

**예제**
```java
// 작성자: John Doe
public void someMethod() {
    // ...
}
```

### 주석으로 처리한 코드
> 주석으로 코드를 비활성화하는 경우로, 버전 관리 시스템을 사용하는 것이 더 적절합니다.

**예제**
```java
// 아래 코드는 나중에 사용할 수 있습니다.
// oldMethod();
```

### HTML 주석
> 코드에 HTML 형식의 주석을 추가하는 경우로, 가독성을 해칩니다.

**예제**
```java
/**
 * <h1>이것은 제목입니다</h1>
 * <p>이것은 설명입니다.</p>
 */
public void myMethod() {
    // ...
}
```

### 전역 정보
> 특정 코드 블록과 관련이 없는 전역적인 정보를 주석으로 추가하는 경우입니다.

**예제**
```java
// 이 모듈은 모든 사용자 정보를 관리합니다.
public void updateUserProfile() {
    // ...
}
```

### 모호한 관계
> 주석과 코드 간의 관계가 명확하지 않은 경우입니다.

**예제**
```java
// 모든 사용자에게 적용됩니다.
public void applyDiscount() {
    // ... (할인 로직)
}
```

### 함수 헤더
> 함수의 목적을 설명하는 주석이지만, 함수 이름과 코드로 충분히 표현 가능한 경우입니다.

**예제**
```java
// 이 함수는 두 수를 더합니다.
public int add(int a, int b) {
    return a + b;
}
```

### 비공개 코드의 JavaDocs
> 비공개(private) 메서드나 클래스에 Javadoc을 추가하는 경우로, 불필요한 문서화입니다.

**예제**
```java
/**
 * 이 메서드는 내부적으로 사용됩니다.
 */
private void internalMethod() {
    // ...
}
```

## 방화벽 (Firewall)
>네트워크 보안을 위한 장치나 소프트웨어로, 네트워크 트래픽을 모니터링하고 제어하여 허용된 트래픽만 통과시키고, 불법적인 접근이나 공격을 차단하는 역할을 함 

### 주요 기능 
- **트래픽 필터링**
	- 방화벽은 패킷의 출발지 IP 주소, 목적지 IP 주소, 포트 번호, 프로토콜 등을 기준으로 네트워크 트래픽을 허용하거나 차단
- **침입 차단**
	- 네트워크를 통해 발생할 수 있는 침입이나 공격을 탐지하고 차단
	- 예를 들어, 해커가 서버에 접근하려는 시도를 막는 역할을 함
- **네트워크 분리**
	- 내부 네트워크와 외부 네트워크(예: 인터넷) 사이에 방화벽을 두어, 내부 네트워크를 외부의 위험으로부터 보호함 
- **로그 기록 및 모니터링**
	- 방화벽은 통과하는 트래픽에 대한 로그를 기록하고, 이를 통해 의심스러운 활동이나 공격 시도를 모니터링할 수 있음 

방화벽은 **하드웨어**로 구현될 수도 있고, **소프트웨어**로 네트워크 운영 체제에서 실행될 수도 있음.


### 하드웨어 vs 소프트웨어 
> 아래는 하드웨어 방화벽과 소프트웨어 방화벽의 차이점임 

| **특징**     | **하드웨어 방화벽**                 | **소프트웨어 방화벽**              |
| ---------- | ---------------------------- | -------------------------- |
| **구성**     | 독립된 물리적 장치                   | 운영 체제에서 실행되는 프로그램          |
| **설치 위치**  | 네트워크 경계에 설치                  | 특정 장치나 서버에 설치              |
| **성능**     | 고성능, 대규모 네트워크에 적합            | 시스템 리소스를 사용하여 성능이 떨어질 수 있음 |
| **관리**     | 전용 관리 콘솔이나 웹 인터페이스를 통해 관리    | 장치 내에서 직접 관리               |
| **비용**     | 비쌈 (하드웨어와 소프트웨어 포함)          | 저렴하거나 무료                   |
| **유연성**    | 이동성 제한, 환경에 맞는 설정이 필요        | 이동 가능, 설정과 업데이트가 용이        |
| **보안 기능**  | 고급 보안 기능 (VPN, IDS/IPS 등 제공) | 기본적인 트래픽 필터링, 일부 고급 기능 제공  |
| **적합한 환경** | 대규모 네트워크, 데이터 센터, ISP 등      | 개인 사용자나 소규모 네트워크, 장치별 보호   |

## Layer 3 & 4 Firewall 
>Layer 3와 Layer 4 방화벽은 OSI 모델의 **네트워크 계층 (Layer 3)**과 **전송 계층 (Layer 4)**에서 트래픽을 필터링하는 방화벽. 둘디 주로 같이 쓰임 

### Layer 3 
- OSI 모델에서 네트워크 계층 (Layer 3)에서 작동함 
- 주로 **IP 주소**(출발지 및 목적지), **서브넷 마스크**, **라우팅 경로** 등을 기준으로 트래픽을 필터링함 
- **예시**
	- IP 주소 기반의 차단, 네트워크 경로 설정에 의한 접근 제한.

```bash 
# 허용: 192.168.1.100 IP에서 오는 모든 트래픽을 허용
iptables -A INPUT -s 192.168.1.100 -j ACCEPT

# 차단: 192.168.1.200 IP로 가는 모든 트래픽을 차단
iptables -A OUTPUT -d 192.168.1.200 -j DROP

# 허용: 192.168.1.0/24 서브넷에서 오는 모든 트래픽을 허용
iptables -A INPUT -s 192.168.1.0/24 -j ACCEPT

# 차단: 192.168.1.100 IP에서 오는 ICMP (ping) 트래픽을 차단
iptables -A INPUT -s 192.168.1.100 -p icmp -j DROP
```

### Layer 4
- OSI 모델에서 전송 계층 (Layer 4)에서 작동합니다. 이 계층은 데이터가 **TCP**(Transmission Control Protocol)와 **UDP**(User Datagram Protocol) 같은 프로토콜을 통해 전송될 때 사용하는 포트 번호를 다룸 
- 주로 **포트 번호**와 **프로토콜**(TCP/UDP)을 기준으로 트래픽을 필터링함
- TCP나 UDP 포트 번호를 이용하여 트래픽을 필터링함 
- **특정 포트**(예: 80번 포트는 HTTP, 443번 포트는 HTTPS)로의 연결을 허용하거나 차단할 수 있음 
- **예시**
	- HTTP 트래픽을 허용하고, FTP 포트를 차단하는 설정.

```bash
# 허용: 80번 포트 (HTTP)에서 들어오는 모든 TCP 트래픽을 허용
iptables -A INPUT -p tcp --dport 80 -j ACCEPT

# 차단: 53번 포트 (DNS)로 나가는 모든 UDP 트래픽을 차단
iptables -A OUTPUT -p udp --dport 53 -j DROP

# 허용: 22번 포트 (SSH)에서 들어오는 TCP 트래픽을 허용
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 허용: 80번 포트 (HTTP)와 443번 포트 (HTTPS)로 들어오는 트래픽을 허용
iptables -A INPUT -p tcp -m multiport --sports 80,443 -j ACCEPT

# 차단: 8080번 포트로 들어오는 모든 트래픽을 차단
iptables -A INPUT -p tcp --dport 8080 -j DROP

```

### 복합적인 규칙 예제 
> 보통 위처럼 따로 사용안하고 Layer3 와 Layer4 를 같이 사용함 
```bash 
# 허용: 192.168.1.100 IP에서 443번 포트 (HTTPS)로 들어오는 TCP 트래픽을 허용
iptables -A INPUT -s 192.168.1.100 -p tcp --dport 443 -j ACCEPT

# 차단: 192.168.1.100 IP에서 25번 포트 (SMTP)로 나가는 모든 트래픽을 차단
iptables -A OUTPUT -s 192.168.1.100 -p tcp --dport 25 -j DROP
```

### Cisco ACL
위에 보여준 예시의 경우 **Linux**에서 사용되는 `iptables` 명령어입니다. `iptables`는 리눅스 기반 방화벽 소프트웨어로, 트래픽을 필터링하는 데 사용됨 

Cisco 장비에 설치된 Cisco IOS의 경우 다른 syntax의 **ACL (Access Control List)** 규칙을 사용함 

아래는 몇가지 예제들임. 
```bash
access-list 101 permit tcp any any eq 80  
# 모든 출발지 IP(any)에서 모든 목적지 IP(any)로 가는 TCP 80번 포트(HTTP) 트래픽을 허용  

access-list 102 permit tcp 192.168.1.100 0.0.0.0 10.10.10.0 0.0.0.255 eq 22 # 출발지 IP: 192.168.1.100 (단일 IP) # 목적지 네트워크: 10.10.10.0/24 # 서비스: TCP 22번 포트 (SSH) # 결과: 192.168.1.100에서 10.10.10.0/24 네트워크로의 SSH 트래픽 허용

access-list 103 deny icmp 192.168.0.0 0.0.255.255 any  
# 출발지 네트워크: 192.168.0.0/16  
# 목적지: any (모든 IP)  
# 프로토콜: ICMP (ping)  
# 결과: 192.168.0.0/16 네트워크에서 오는 모든 ping 트래픽 차단  

access-list 104 deny tcp any any eq 20  
# 포트 20(FTP 데이터) 트래픽 차단  
access-list 104 deny tcp any any eq 21  
# 포트 21(FTP 명령) 트래픽 차단  

access-list 105 permit tcp any any range 1000 2000  
# 출발지와 목적지 관계없이 1000번에서 2000번 사이의 TCP 포트 트래픽을 허용  

```

## Level 7 Firewall 
> **OSI 7계층 중 응용 계층(Application Layer)** 에서 동작하는 고급 방화벽으로 흔히 차세대 방화벽 (NGFW: Next-Generation Firewall)이라고도 불림   
>기존의 전통적인 방화벽이 3계층(네트워크 계층)과 4계층(전송 계층)에서 IP 주소와 포트를 기준으로 트래픽을 필터링하는 반면, NGFW는 **애플리케이션, 사용자, 데이터 내용**까지 심층적으로 분석하고 관리할 수 있음. 
### 주요기능 
- **애플리케이션 인식 및 제어**
	- 특정 애플리케이션(예: 페이스북, 유튜브, 스카이프 등)을 인식하고 트래픽을 관리.
	- 예: 페이스북 접속은 허용하지만 페이스북 채팅 기능은 차단.
- **심층 패킷 검사 (DPI, Deep Packet Inspection)**
	- 패킷의 헤더뿐만 아니라 내용(payload)을 분석하여 악성 콘텐츠, 공격, 또는 비인가 데이터 전송을 탐지.
    - 네트워크를 통해 전달되는 실제 데이터를 검사.
- **사용자 기반 정책**
	- Active Directory, LDAP 같은 인증 시스템과 통합하여 **특정 사용자나 그룹**에 대한 트래픽 제어.
	- 예: '게스트 그룹'은 소셜 미디어 차단, '마케팅 그룹'은 허용.
- **통합 침입 방지 시스템(IPS, Intrusion Prevention System)**
	- 기존 방화벽과 달리 IPS 기능을 통합하여 악성 코드, 랜섬웨어, 제로데이 공격 등을 탐지 및 차단.
	- 내부와 외부 위협 모두 보호.
- **SSL/TLS 암호화 트래픽 검사**
	- HTTPS와 같은 암호화된 트래픽을 해독하여 숨겨진 악성 활동이나 데이터 유출 시도를 탐지.
	- 검사 후 트래픽을 다시 암호화하여 보안 유지.
- **고급 위협 방어**
    - 샌드박싱(Sandboxing)을 통해 의심스러운 파일이나 코드를 격리된 환경에서 실행하여 악성 여부를 판단.
    - 최신 위협 정보를 실시간으로 업데이트해 보호.
- **응용 계층 로깅 및 보고**
    - 애플리케이션 사용 현황, 사용자 행동, 보안 사건 등을 세부적으로 기록하고 보고.
    - 감사 및 포렌식 조사에 유용.
- **콘텐츠 필터링 및 데이터 유출 방지(DLP, Data Loss Prevention)**
    - 웹 콘텐츠 카테고리(예: 성인 콘텐츠, 도박, 소셜 미디어 등) 기반으로 접근을 제한.
    - 신용카드 정보나 기밀 데이터를 외부로 유출하는 것을 방지.
- **클라우드 통합**
    - 하이브리드 환경에서 클라우드 보안 솔루션 및 SD-WAN과 원활하게 통합

### 기존 방화벽과 차이점 

| **기능**            | **전통적 방화벽** | **차세대 방화벽(NGFW)**   |
| ----------------- | ----------- | ------------------- |
| **동작 계층**         | OSI 3/4계층   | OSI 7계층 (애플리케이션 계층) |
| **애플리케이션 인식**     | 없음          | 있음                  |
| **사용자 기반 정책**     | 없음          | 있음                  |
| **침입 방지 기능(IPS)** | 별도 장비 필요    | 통합 제공               |
| **암호화 트래픽 검사**    | 없음          | 있음                  |
| **위협 정보 실시간 통합**  | 없음          | 있음                  |

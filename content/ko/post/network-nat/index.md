## 배경지식 
IP 주소의 경우 Public IP Address 와 Private IP Address로 나누어져 있음 

아래 Private IP Address의 경우 global하게 유니크 하지 않음. 내부 네트워크에서만 사용되어야함. 
- 10.0.0.0 - 10.255.255.255
- 172.16.0.0 - 172.31.255.255
- 192.168.0.0 - 192.168.255.255 

이외에 주소들만 gloabl 하게 고유한 public IP Address로 사용될수 있음. 

그럼 저 Private IP주소들은 어케 접속하는 거임? -> **NAT** 

## NAT (Network Address Translation)
>네트워크 내에서 IP 주소를 변환하는 기술. 주로 하나의 공인 IP 주소로 여러 내부 장치가 인터넷에 접속할 수 있게 해줌 

### 주요 기능 
1. **IP 주소 절약**
	- 내부 네트워크에서 여러 장치가 하나의 공인 IP 주소를 공유하여 인터넷에 연결되기 때문에, 공인 IP 주소를 절약할 수 있습니다.
2. **보안**
	- NAT는 내부 네트워크의 구조를 숨길 수 있어 외부에서 내부 네트워크에 대한 직접적인 접근을 어렵게 만들어 보안상 이점을 제공합니다.
3. **인터넷 접근 관리** 
	- NAT는 인터넷과의 연결을 관리하며, 특정 포트 번호를 기반으로 트래픽을 다시 내부 네트워크로 라우팅할 수 있습니다.

### 그럼 내부 IP 구별은 어케하는거임? 
**포트 주소 변환(PAT)** 을 사용해서 함.
1. **내부 장치에서 외부로 나갈 때**:
	- 내부 네트워크의 장치들이 인터넷에 접근할 때, 각각의 요청은 내부 IP 주소와 포트 번호를 가진 상태로 나감 
	- NAT 장치는 각 요청에 대해 **공인 IP 주소와 고유한 포트 번호**를 할당합니다.
	- 예를 들어, 내부 IP `192.168.1.2`의 포트 `12345`에서 나가는 트래픽은 공인 IP `203.0.113.1`의 포트 `10001`로 변환됨 
2. **응답이 들어올 때**:
	- 외부에서 들어오는 응답은 공인 IP 주소 `203.0.113.1`의 포트 `10001`로 전달됩니다.
	- NAT 장치는 이 응답이 어떤 내부 장치로 돌아가야 하는지 확인하기 위해 **포트 번호**를 기준으로 내부 장치와 매핑된 정보를 사용합니다.
	- 예를 들어, `203.0.113.1:10001`로 들어오는 응답은 내부 IP `192.168.1.2`의 포트 `12345`와 연결된 요청에 대한 응답임을 알아내고, 해당 응답을 내부 네트워크의 `192.168.1.2` 장치로 전달합니다.

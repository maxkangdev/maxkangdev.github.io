## WAN (Wide Area Network)

광역 네트워크를 의미하며, 넒은 지리적 범위를 아우르는 네트워크 
- 주로 여러개의 LAN 을 연결함
- 회사의 지사나, 도시, 나라, 심지어 대륙간의 통신을 가능케 함 

### 특징 
- 공용 또는 사설 네트워크
	- 공공 통신망(인터넷 등) 이나 전용 임대 회선을 사용하여 네트워크를 구성함 
- 통신 매체
	- 광섬유, 위성 링크, MPLS 등의 기술을 활용함 
- 분산 소유권
	- 여러 조직이나 서비스 제공자가 소유한 인프라로 구성되는 경우가 많음 

### 예시
 - 인터넷 
 - 기업 네트워크
	 - 큰 기업이 전 세계 지사를 연결하기 위해 사용하는 네트워크
 - 통신 네트워크
	 - 통신 회사가 제공하는 인터넷 및 전화 서비스를 지원하는 네트

### 장단점 

|                                            장점                                             |                                                       단점                                                       |
| :---------------------------------------------------------------------------------------: | :------------------------------------------------------------------------------------------------------------: |
| - 전 세계적으로 연결성과 통신이 가능함.<br>- 장거리에서 리소스와 정보를 공유할 수 있음.<br>- 다국적 기업이 중앙 집중식 데이터 관리를 가능하게 함. | - 구축 및 유지 관리 비용이 상대적으로 높음.<br>- 장거리 통신이나 공유 인프라로 인해 지연 시간과 신뢰성 문제가 발생할 수 있음.<br>- 데이터 전송 중 보안을 강화해야 하는 필요성이 큼. |

## 인터넷 연결 

![](Screenshot%202025-01-23%20at%209.55.44%20AM.png)

먼저 각 컴포넌트들이 무슨역할을 하는지 알아봅시다 
(Router 부터 아래까지는 [[Network]Router](network-router) 글을 읽고 오면 대충 알거임)

### ISP (Internet Service Provider)
- 인터넷 연결을 제공하는 회사임
- 가정이나 회사의 네트워크를 전 세계 인터넷과 연결해  (예: KT, SK브로드밴드, LG U+)

### Border Router 
- 내부 네트워크를 ISP의 네트워크와 연결하는 장치.
- 주요 역할:
    - 내부 네트워크와 인터넷 간의 데이터 전송을 관리함 
    - NAT(Network Address Translation) 또는 방화벽 기능을 통해 보안을 강화함 

### **Ethernet Switch (경계 라우터 근처의 오렌지색 스위치)**
- 경계 라우터에서 데이터를 받아 내부 네트워크로 전달하는 스위치
- 스위치는 네트워크 상에서 데이터의 목적지(MAC 주소 기반)를 확인하고 해당 장치로 데이터를 전달함 
- 여기서는 네트워크를 여러 하위 네트워크로 나누는 중심 역할을 함 

### 라우터가 왜 두개나 필요함? 
- **첫 번째 라우터(경계 라우터)** 👉 **외부와 내부를 연결하는 역할**을 담당함 
	- 인터넷 서비스 제공업체(ISP)와 내부 네트워크를 연결함 
    - 이 라우터는 외부 인터넷 트래픽을 처리하고, NAT(Network Address Translation)를 통해 외부 IP 주소를 내부 네트워크의 여러 장치(IP)와 연결합니다.  
- **두 번째 라우터(내부 라우터)** 👉 **내부 네트워크 간의 데이터 전송**을 처리함 
	- 내부 네트워크의 다양한 하위 네트워크를 연결함. 
	- 예를 들어, 여러 층의 사무실이나 부서별 네트워크가 분리되어 있다면, 이 라우터가 각 네트워크 간 데이터 흐름을 관리함 
#### 요약
- 경계 라우터: 외부(인터넷) ↔ 내부(로컬 네트워크) 연결
- 내부 라우터: 내부 네트워크 ↔ 내부 네트워크 연결

### 스위치로 왜 두개나 필요함?
- **첫 번째 스위치(오렌지색)**:  
	- 경계 라우터와 내부 라우터를 연결함
    - 이 스위치는 "백본(backbone)" 역할을 하며, 외부 인터넷에서 내부 네트워크로 들어오는 데이터를 내부 라우터로 전달함
    - 대규모 네트워크에서는 이런 백본 스위치가 필수적
- **두 번째 스위치들(파란색)**:  
	- 내부 라우터와 여러 컴퓨터를 연결함 
    - 각각의 스위치가 여러 장치를 연결하여 네트워크를 확장함 
    - 대규모 네트워크에서 물리적으로 가까운 장치들을 하나의 스위치에 연결하면 데이터 전송 속도와 효율성이 높아짐
#### 요약
- **첫 번째 스위치**: 외부(경계 라우터)와 내부(내부 라우터)를 연결
- **두 번째 스위치들**: 내부 라우터와 여러 컴퓨터를 연결


## VPN (Virtual Private Network) 
WAN만 있으면 다 되는거아님? VPN은 또 뭐고 왜필요한거임? 

### WAN 문제점 
- **인터넷을 경유하지 않는 WAN의 경우**
	- 전용회선 (MLPS, 전용 광섬유 회선)을 사용하는 WAN의 경우 물리적인 인프라를 구성해야함. 
		- 개비쌈 + 유지 관리 비용 높음 + 커질수록 기하급수적으로 높아짐 
		- 인프라 + 유지보수 인력 비용 + 관련 장비 +  관련 소프트웨어 비용... 
	- 결과적으로 돈먹는 하마임.
- **인터넷을 사용해서 WAN을 구축하는 경우** 
	- 설치 및 운영 비용이 더쌈 
	- 근데 느림... 
	- 그리고 보안도 별로임... 

그래서 등장한게 -> **VPN** 

### 목적
 - 공용 네트워크 (인터넷)을 통해 **보안된** 연결을 제공하는 것 
 - 인터넷을 통해 공공 wifi 를 사용하거나, 데이터 암호화, IP 주소 숨김, 또는 지역 컨텐츠 우회를 위해 사용됨 


## IPSEC VPN 

**IPsec VPN**은 **IP Security (IPsec)** 프로토콜을 사용하여 **인터넷과 같은 공용 네트워크**를 통해 **안전하게 데이터를 전송**하는 **VPN 기술**

### 동작방식 
IPsec VPN은 **터널 모드**(Tunnel Mode)와 **전송 모드**(Transport Mode) 두 가지 모드로 동작함 
- **터널 모드 (Tunnel Mode)**:  
    - 가장 일반적으로 사용되는 방식으로, IPsec은 **전체 패킷**을 암호화하고, 암호화된 패킷을 새로운 헤더로 캡슐화하여 전송함. 
    - 이 방식은 **두 네트워크 간의 연결**(예: 원격 근무자와 회사 네트워크)에서 주로 사용됨 
- **전송 모드 (Transport Mode)**:  
	- 데이터 자체만 암호화하고 IP 헤더는 변경하지 않음 
	  - 이 모드는 일반적으로 **호스트 간의 보안 연결**에서 사용됨 

| 항목           | **터널 모드 (Tunnel Mode)**              | **전송 모드 (Transport Mode)**            |
| ------------ | ------------------------------------ | ------------------------------------- |
| **암호화 범위**   | 전체 패킷 (헤더 + 데이터)                     | 데이터만 (헤더는 그대로)                        |
| **헤더 변경 여부** | 새로운 IP 헤더 추가 (패킷 캡슐화)                | 기존 IP 헤더를 그대로 사용                      |
| **사용 목적**    | 두 네트워크 간의 연결 (예: 원격 근무자와 회사 네트워크 연결) | 두 호스트 간의 보안 통신 (예: 서버 간의 데이터 전송)      |
| **보안성**      | 더 높은 보안 (전체 패킷을 암호화하여 모든 정보를 보호)     | 상대적으로 낮은 보안 (헤더가 암호화되지 않음)            |
| **주요 사용 예시** | 사이트 간 VPN, 원격 근무자 연결                 | 서버 간 보안 통신, 엔드 투 엔드 보안 연결             |
| **패킷 크기**    | 패킷에 추가 헤더가 붙기 때문에 **패킷 크기가 커질 수 있음** | 헤더는 그대로 유지되어 **패킷 크기 변화가 적음**         |
| **성능**       | 암호화와 캡슐화로 인해 상대적으로 성능이 **저하될 수 있음**  | 패킷 암호화 범위가 제한적이므로 성능 저하가 상대적으로 **적음** |
> 헤더에는 목적지 주소, 출발지 주소 라우팅 정보 등이 들어 있어서 해당 부분 까지 암호화 해버리면 위치 추적 방지, 익명성 보호, 패킷 분석 방지, 중간자 공격 방어 등이 가능해서 ㄹㅇ 로 터널을 뚫는 거라고 볼수 있음. 


## Layer 2 VPN 
**OSI 모델의 데이터 링크 계층**(Layer 2)에서 동작하는 **가상 사설망*임.  

즉, Layer 2 VPN은 **물리적인 네트워크 상에서 마치 동일한 로컬 네트워크**처럼 여러 지점(서버, 사이트 등)을 연결하여, **물리적인 거리가 떨어져 있어도 네트워크 상의 장비들이 동일한 네트워크에 있는 것처럼** 작동하게 만듬

쉽게 말해서 한국에 있는 지사와 미국에 있는 지사가 같은 네트워크 subnet을 가지게 할 수 있음 
- w/o VPN
	- 10.1.1.x(한국) 
	- 10.1.2.x(미국)
- with VPN
	- 10.1.1.x(한국)
	- 10.1.1.x(미국)

### 특징
 - **데이터 링크 계층에서 동작**:
    - Layer 2 VPN은 **데이터 링크 계층(2계층)**에서 동작하므로, **이더넷**이나 **프레임 릴레이**, **ATM** 등을 사용하여 연결을 설정
    - 실제 네트워크에서는 **MAC 주소**를 사용해 데이터를 전송하며, 네트워크가 서로 다른 물리적 위치에 있더라도 **로컬 네트워크처럼 동작**할 수 있게 
- **캡슐화**
    - Layer 2 VPN에서는 **이더넷 프레임** 또는 **데이터 링크 계층의 프레임**을 다른 네트워크로 전달하기 위해 캡슐화함
    - 이 캡슐화는 Layer 3 VPN과는 달리, IP 헤더를 다루지 않고, **데이터 링크 계층**에 필요한 정보만 캡슐화함
- **투명성**
    - **사용자 네트워크**가 **물리적으로 분리**되어 있어도 **동일한 LAN**처럼 작동하므로, **애플리케이션 계층에서의 변화가 없음**. 
    - 예를 들어, 네트워크 간에 **VLAN**을 그대로 전달할 수 있음

### **Layer 2 VPN의 사용 사례**
1. **사이트 간 연결 (Site-to-Site VPN)**
    - 물리적으로 떨어진 두 지점(예: 두 데이터 센터나 두 회사 사무실)을 **같은 LAN**처럼 연결할 때 사용됨
    - 예를 들어, 지점 A에서 보내는 **이더넷 프레임**이 지점 B까지 그대로 전달되며, B는 이를 받아서 로컬 네트워크처럼 사용함 
2. **서비스 제공업체의 VPN 서비스**
    - **기업**이 여러 지점에 분산된 네트워크를 연결하고자 할 때, ISP(인터넷 서비스 제공업체)가 **Layer 2 VPN** 서비스를 제공하여 **기업 네트워크**를 여러 지점 간 연결하는 데 사용함.
3. **VLAN 간의 연결**
    - **VLAN**을 **가상 네트워크**로 전송할 수 있도록 하여, 서로 다른 VLAN을 물리적 거리가 떨어져 있어도 연결할 수 있게 만듬 
> What is VLAN? [[Network]VLAN](network-vlan)

### 장단점 

|**항목**|**장점**|**단점**|
|---|---|---|
|**투명한 네트워크 연결**|- 서로 다른 지점 간에 **같은 LAN**처럼 동작. **애플리케이션에 영향 없음**|- **구성**과 **관리**가 더 복잡할 수 있음.|
|**기존 인프라 활용**|- 기존 **이더넷 네트워크** 또는 **VLAN**을 그대로 사용 가능.|- **대역폭 소비**가 많아질 수 있음.|
|**보안**|- **캡슐화**와 **암호화** 기능을 통해 **보안 강화**.|- **패킷 캡슐화**로 인해 **성능 저하**가 발생할 수 있음.|
|**사이트 간 연결**|- **멀리 떨어진 지점**을 물리적으로 연결하지 않고도 **로컬 네트워크처럼 연결**|- **다수의 지점 연결 시 관리**가 어려워질 수 있음.|
|**확장성**|- **MPLS 기반 L2VPN**은 **확장성**이 뛰어남.|- **복잡한 설정**이 요구될 수 있음.|

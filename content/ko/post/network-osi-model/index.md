## OSI Model 

 - 컴퓨터 네트워크의 통신 과정을 7개의 계층으로 나눈 모델
 - 이 모델은 네트워크 통신을 이해하고 문제를 진단하는 데 유용한 이론적인 구조로, 각 계층은 네트워크에서 발생하는 특정 기능을 담당함 
 - 네트워크 프로토콜과 장치들이 어떻게 상호작용하는지 설명하는 데 사용됨 

### Summary Table 

| 계층 번호 | 계층 이름                       | 기능                                | 전송 단위 | 예시 프로토콜               |
| ----- | --------------------------- | --------------------------------- | ----- | --------------------- |
| 7     | 응용 계층 (Application Layer)   | 사용자와 직접 상호작용하는 계층, 애플리케이션 서비스 제공  | 메시지   | HTTP, FTP, SMTP       |
| 6     | 표현 계층 (Presentation Layer)  | 데이터 형식 변환, 압축, 암호화 및 인코딩/디코딩      | 메시지   | JPEG, GIF, SSL, TLS   |
| 5     | 세션 계층 (Session Layer)       | 통신 세션 설정, 관리 및 종료, 통신 흐름 제어       | 메시지   | NetBIOS, RPC          |
| 4     | 전송 계층 (Transport Layer)     | 데이터의 신뢰성 보장, 흐름 제어, 오류 수정         | 세그먼트  | TCP, UDP              |
| 3     | 네트워크 계층 (Network Layer)     | 데이터 라우팅, IP 주소 관리, 패킷 전송          | 패킷    | IP, ICMP, IPX         |
| 2     | 데이터 링크 계층 (Data Link Layer) | 데이터 프레임 전송, 오류 감지 및 수정, MAC 주소 관리 | 프레임   | Ethernet, PPP, Switch |
| 1     | 물리 계층 (Physical Layer)      | 물리적 매체를 통한 비트 전송, 전기적/광학적 신호 전송   | 비트    | 케이블, 리피터, 허브          |

### 7. **응용 계층 (Application Layer)**
- **기능** 
	- 사용자와 직접 상호작용하는 계층
	- 데이터가 네트워크를 통해 전송될 수 있도록 애플리케이션 간에 서비스를 제공함. 
	- HTTP, FTP, SMTP 등과 같은 프로토콜이 이 계층에 포함됨 
- **장비 예시**: 없음 (소프트웨어 계층)
- **전송 단위**: 메시지
### 6. 표현 계층 (Presentation Layer)
- **기능** 
	- 데이터의 표현 방식을 변환함. 
	- 예를 들어, 데이터의 압축, 암호화, 인코딩/디코딩 등을 처리합니다. 
	- 주로 데이터를 응용 계층에서 사용할 수 있도록 형식을 맞추는 역할을 합니다.
- **장비 예시**: 없음 (소프트웨어 계층)
- **전송 단위**: 메시지
### 5. **세션 계층 (Session Layer)**
- **기능** 
	- 통신 세션을 설정, 관리, 종료함
	- 통신이 끊어지지 않도록 세션을 유지하고, 여러 응용 프로그램 간의 데이터 교환을 관리함 
- **장비 예시**: 없음 (소프트웨어 계층)
- **전송 단위**: 메시지
### 4. **전송 계층 (Transport Layer)**
- **기능** 
	- 데이터의 신뢰성을 보장하고, 흐름 제어 및 오류 수정을 수행함. 
	- 송신 측과 수신 측의 포트를 통해 **세그먼트** 단위로 데이터를 전달하며, 데이터의 순서나 정확성을 확인함 
- **장비 예시**: 없음 (소프트웨어 계층)
- **전송 단위**: 세그먼트 (TCP) / 데이터그램 (UDP)
### 3. **네트워크 계층 (Network Layer)**
- **기능**
	- 데이터를 목적지까지 전달하기 위한 경로를 선택하고, 패킷을 라우팅함. 
	- 네트워크 간의 연결을 설정하고, **IP 주소**를 사용하여 네트워크 간 통신을 수행힘 
- **장비 예시**: 라우터
- **전송 단위**: 패킷
### 2. **데이터 링크 계층 (Data Link Layer)**
- **기능**
	- 물리적 전송 오류를 수정하고, 데이터의 전송을 안정화함 
	- 네트워크의 장치 간에 데이터 프레임을 전달하며, **MAC 주소**를 사용하여 장치 간 통신을 관리함 
- **장비 예시**: 스위치, 브리지
- **전송 단위**: 프레임
### 1. **물리 계층 (Physical Layer)**
-  **기능**
	- 데이터 전송을 위한 물리적 매체를 제공함. 
	- 전기적 신호, 광 신호, 라디오파 등을 사용하여 비트(0과 1)를 전송함.
- **장비 예시**: 케이블, 허브, 리피터, 네트워크 인터페이스 카드(NIC), 스위치의 포트 등
- **전송 단위**: 비트

## 개요
네트워크에 대한 지식이 부족하다 보니 개발시 항상 네트워크부분에서 시간이 소요됨.
이김에 네트워크에 대해서 확실하게 정리해서 헷갈리는 부분을 없애버려야겠음 🤓.

## LAN  (Local Area Network)
- 비교적 제한된 범위 내에서 연결된 컴퓨터와 기타 장치들 간의 네트워크를 의미함
- 주로 가정, 사무실, 학교와 같은 작은 규모의 공간에서 사용됨 
- 장치 간의 데이터 전송을 빠르고 효율적으로 처리할 수 있도록 설계됨. 

### 구성요소 
- **네트워크 장비**
	-  **스위치**: 여러 장치를 하나의 LAN으로 연결.
	- . **라우터**: LAN을 외부 네트워크(인터넷)와 연결.
	- . **액세스 포인트(AP)**: 무선 LAN(WLAN)을 위한 장치.
- **네트워크 케이블**:
	- 유선 LAN에서는 이더넷 케이블이 사용됨
- **네트워크 어댑터**
	- 각 장치에 LAN에 연결하기 위한 네트워크 인터페이스 카드(NIC)가 필요함

>각 컴퓨터마다 NIC가 달려있음. 그리고 NIC는 요렇게 생겼음.   
![](Screenshot%202025-01-19%20at%2011.44.57%20AM.png)
### 종류
- **유선 LAN**:
    - 이더넷 케이블을 사용해 장치를 물리적으로 연결.
    - 안정적이고 빠른 성능 제공.
- **무선 LAN (WLAN)**:
    - Wi-Fi를 통해 무선으로 장치를 연결.
    - 이동성과 편의성이 높음.

## Mac Address (Media Access Control Address )

물리적으로 네트워크에서 장치를 식별할때 사용됨. 
### 구조
- 일반적으로 **48비트(6바이트)** 길이로 구성되며, 16진수로 표현됨
	- 보통 **콜론(:)** 또는 **하이픈(-)** 으로 구분된 12자리의 16진수 문자열
	- 처음 3바이트는 네트워크 장치를 제조한 회사의 고유 식별자
		- 아래 `00:14:22`는 특정 제조업체에 할당된 식별자임
	- 나머지 3바이트는 각 장치에 대해 고유하게 할당됩니다. 즉, 각 장치마다 고유한 MAC 주소가 있음 
	- Ex. `00:14:22:01:23:45` 또는 `00-14-22-01-23-45`

### 용도
-  **네트워크 장치 식별**
	- NIC의 포트 하나당 Mac Address가 하나 할당됨. 포트당 유니크한 값이 할당된다고 보면됨. 
	- MAC 주소는 네트워크 상에서 장치를 고유하게 식별하는 데 사용됨 
- **패킷 전달**
	- 데이터 링크 계층에서 사용되며, 패킷을 목적지 장치로 전달하기 위한 주소로 활용됩니다. 네트워크 장치가 다른 장치와 통신할 때, MAC 주소를 사용하여 데이터가 올바른 장치로 전달되도록 함. 
- **네트워크 보안**
	- 일부 네트워크 보안 시스템에서는 MAC 주소를 기반으로 장치 접근을 제한하거나, MAC 주소 필터링을 통해 네트워크에 연결할 수 있는 장치를 제어함
- **DHCP 할당**
	- 네트워크에서 **DHCP(동적 호스트 구성 프로토콜)**를 사용할 때, MAC 주소를 통해 IP 주소를 동적으로 할당받을 수 있음

### Mac vs IP
- **MAC 주소**: 네트워크 장치의 하드웨어 고유 주소이며, 장치가 인터넷에 연결될 때마다 변경되지 않음
- **IP 주소**: 네트워크에서 장치를 식별하는 논리적 주소로, 네트워크 환경에 따라 동적으로 변경될 수 있음
> 따라서 **MAC 주소**는 로컬 네트워크에서 장치를 구분하고, **IP 주소**는 인터넷에서 장치를 구분하는 데 사용됨 

### 그럼 노트북에는 Mac Address 없음? Port 없으니까? 
ㄴㄴ NIC는 이더넷만 있는게 아님. 와이파이용, 블루투스용 등 다 있음. 노트북에도 와이파이, 블루투스 용 NIC 있으니까 Mac Address 도 있음.

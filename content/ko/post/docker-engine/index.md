## Docker 란?
> **"컨테이너"** 라는 기술을 사용해 애플리케이션과 그 실행 환경을 하나의 패키지로 묶어서 어디서나 쉽게 실행할 수 있게 해주는 도구 (Programmed in Go)

### 쉽게 설명하자면
- 애플리케이션을 실행하려면 보통 **코드, 라이브러리, 설정, 데이터베이스** 같은 여러 가지가 필요함 
- 그런데, 이 환경을 직접 구성하다 보면 운영 체제마다 다르고, 개발자마다 다르고, 설정이 꼬여서 **"내 컴퓨터에서는 잘 되는데?"** 같은 문제가 자주 생김 😓

>**도커 등장**  

도커는 애플리케이션과 그 실행에 필요한 모든 걸 "컨테이너" 라는 박스에 담아둠. 이렇게 되면 
1. **내 컴퓨터**에서 실행하든,
2. **팀원 컴퓨터**에서 실행하든,
3. **서버**나 **클라우드**에서 실행하든,  
    모두 **똑같이 작동**함 😤

## 비유 
음식을 요리하려면 재료도 사고, 조리법도 알아야 하고, 주방도 있어야 하잖슴? 

도커는 **완벽히 조리된 도시락**을 주는 거임 
- 어디서나 그 도시락을 꺼내서 바로 먹을 수 있음 
- 심지어 도시락을 만드는 방법(도커 이미지)도 있으니까 필요하면 새로 만들 수도 있음 

## 도커 아키텍쳐 
> 클라이언트 - 서버 아키텍쳐를 사용함 

도커를 설치한다는 것은 아래 세개의 Component를 설치한다는 것과 동일함
- Docker CLI `docker`
- Rest API 
- Docker Daemon `dockerd`

Docker CLI는 Docker Rest API를 사용해서 Docker Daemon과 통신함.   
그리고 요청을 받은 Docker Daemon 이 사실상 이미지, 컨테이너, 네트워크, 볼륨등의 도커 오프젝트들을 생성하거나 없애는등의 작업을 모두 수행함. 

>**Note**  
>Docker CLI의 경우 굳이 같은 환경에 있을 필요 없음. Remote 설정을 통해 다른 호스트에 설치된 도커 엔진 (API + Daemon) 을 호출 할 수 있음. 
>Ex. `docker -H=remote-docker-engine:1234`
